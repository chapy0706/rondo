# CLAUDE.md

このリポジトリ（rondo）では、Claude Code を設計補助・実装補助として使用します。

## 基本方針

Claude Code は実装者ではなく、設計意図を守る共同作業者として扱います。

実装前に必ず以下を確認してください。

1. README.md
2. 着手対象の docs/issues/*.md（status: open のもの）
3. docs/adr/*.md
4. .claude/rules/*.md
5. 既存のディレクトリ構成

## 作業ルール

- 仕様が曖昧なまま実装しない
- Issue に記載のない設計判断が必要になったら、実装を止めて人間に確認する
- 依存方向を崩さない（Domain を中心に、外側から内側へ依存する）
- Domain 層にフレームワーク・DB・認証・環境変数を持ち込まない
- 境界（外部入力）では unknown を検証してから扱う

## やってはいけないこと

- Makefile を勝手に書き換える（検証の定義であり、信頼境界のため）
- .claude/ 配下（hooks, verify.sh, settings.json）を勝手に書き換える
- ログ系テーブルに対し、終了時刻の確定以外の UPDATE・DELETE を行う
- 環境変数をコードに直書きする
- ドキュメントに絵文字を入れる（プロジェクト全体で絵文字なしに統一）

## 完了条件

コードを変更したら、検証（.claude/verify.sh）が通る状態にしてからターンを終えること。
検証は Stop hook により自動実行される。失敗した場合は修正してから完了とする。

## Claude Code への期待

- 実装前に影響範囲を説明する（安全性 / 変更容易性 / 性能 / 運用 の観点で）
- ファイル変更理由を明確にする
- セキュリティ・認証・プライバシーのリスクを指摘する
- 不要な抽象化を増やさない
