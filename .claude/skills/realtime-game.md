# スキル: リアルタイムゲームの追加

## いつ使うか

複数人がルームに集まり、サーバーを介して同期するゲーム（Tilt Maze 等）を追加するとき。クライアントとサーバーの両方を実装する。

## 原則

- RealtimeGame として実装する（ADR 0004）
- 入力は VirtualPad を使う（ADR 0018）
- 物理や演出はクライアントで回してよいが、勝敗・順位はサーバーが決める
- 順位はサーバーが受信した順を正とする。クライアントのタイムスタンプは使わない（ADR 0014）
- ゲーム固有の権威判定は server/src/games/<id>/ に置く（ADR 0008）

## 手順

クライアント側:

1. apps/web/src/games/<id>/ を作る
2. manifest.ts を書く（kind: "realtime"）
3. game-sdk の useRealtimeGame と useVirtualPad で実装する
4. registry に登録する

サーバー側:

5. server/src/games/<id>/ にゲーム固有の権威ロジックを置く
6. ゴール到達などのイベントを受信順に処理し、順位を確定する

## パターン

クライアント（接続は SDK 経由、決定権を持たない）:

```tsx
const { x, y } = useVirtualPad();
const { send, on } = useRealtimeGame(tiltMazeManifest);

on("rank", (rank) => { /* サーバーが確定した順位を表示 */ });

// ゴール到達。順位はサーバーが決めるので通知のみ
send({ type: "goal" });
```

サーバー（受信順で順位を確定。ADR 0014）:

ゴール到達メッセージを受け取った順にプレイヤーを並べ、それを順位とする。サーバーの受信時刻が唯一の基準であり、メッセージに含まれるクライアント時刻は参照しない。

## やってはいけないこと

- クライアントのタイムスタンプで順位を決める（ADR 0014 違反）
- クライアントに勝敗の決定権を持たせる
- 権威ロジックを基盤側に書く。ゲーム固有の判定は server/src/games/<id>/ に閉じる
