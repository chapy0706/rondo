# issue-16: ソロゲーム 2048

## 背景

ソロゲームの候補のひとつ。4x4 の盤面でタイルをスライドさせ、同じ数字を合成して 2048 を目指すパズル。

VirtualPad の4方向入力だけで完結し、盤面のロジックも素直なため、プラグインとしてのゲーム追加を学ぶ題材として扱いやすい。基盤との接点はマニフェストとスコア返却のみで、Gleam に触れる必要がない。

## スコープ

apps/web/src/games/2048 に、ソロゲームとして 2048 を実装する。

### このissueでやること

- 4x4 盤面のスライドと合成ロジック
- 新しいタイルの出現（2 または 4）
- スコア計算
- 終了判定（動かせる手がなくなったら終了）
- マニフェストの作成と registry への登録
- VirtualPad による4方向入力
- スコアの基盤への返却

### このissueでやらないこと

- リアルタイム要素・通信
- スコアの永続化（Phase 2）
- アンドゥ機能

## 設計方針

- SoloGame として実装し、リアルタイム接続を受け取らない（ADR 0004）
- 入力は基盤共通の VirtualPad を使う。4方向の方向入力のみで操作する（ADR 0018）
- マニフェストで自己記述し、registry に登録する（ADR 0003）
- 盤面のスライド・合成ロジックは判定ロジックであり、テストファーストで書く
- 追加にあたり基盤側のコードには手を入れない

## 受け入れ条件

- ゲーム選択画面から起動して遊べる
- VirtualPad の4方向でタイルがスライドし、同じ数字が合成される
- 動かせる手がなくなると終了し、スコアが基盤に返る
- 盤面ロジックのテストが通る
- 追加にあたり基盤側のコードを変更していない
- 1ゲームが2〜3分程度で終わる想定に収まっている

## 段階

1. マニフェストを作り registry に登録する
2. 盤面のスライド・合成ロジックをテストファーストで実装する
3. 新タイル出現と終了判定を実装する
4. VirtualPad 入力と描画を繋ぐ
5. スコア返却を繋ぐ
