# issue-17: ソロゲーム Snake

## 背景

ソロゲームの候補のひとつ。蛇を操作して餌を食べ、体を伸ばしながら壁と自分自身を避け続ける。

ルールが広く知られており、実装に必要なコード量が最も少ない。VirtualPad の4方向だけで完結する。ソロゲームの追加をひととおり通す最初の一本として、最も軽い選択肢になる。

## スコープ

apps/web/src/games/snake に、ソロゲームとして Snake を実装する。

### このissueでやること

- 蛇の移動と成長
- 餌の出現と取得
- 衝突判定（壁・自分自身）
- スコア計算（食べた餌の数）
- マニフェストの作成と registry への登録
- VirtualPad による4方向入力
- スコアの基盤への返却

### このissueでやらないこと

- リアルタイム要素・通信
- スコアの永続化（Phase 2）
- 難易度選択・ステージ

## 設計方針

- SoloGame として実装し、リアルタイム接続を受け取らない（ADR 0004）
- 入力は基盤共通の VirtualPad を使う。4方向の方向入力のみで操作する（ADR 0018）
- マニフェストで自己記述し、registry に登録する（ADR 0003）
- 衝突判定は判定ロジックであり、テストファーストで書く
- 追加にあたり基盤側のコードには手を入れない

## 受け入れ条件

- ゲーム選択画面から起動して遊べる
- VirtualPad の4方向で蛇の進行方向を変えられる
- 餌を食べると蛇が伸び、スコアが増える
- 壁または自分自身に衝突すると終了し、スコアが基盤に返る
- 衝突判定のテストが通る
- 追加にあたり基盤側のコードを変更していない
- 1ゲームが2〜3分程度で終わる想定に収まっている

## 段階

1. マニフェストを作り registry に登録する
2. 蛇の移動と成長をテストファーストで実装する
3. 餌の出現・取得と衝突判定を実装する
4. VirtualPad 入力と描画を繋ぐ
5. スコア返却を繋ぐ
