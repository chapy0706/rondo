# issue-12: Tilt Maze クライアント

## 背景

Tilt Maze は Phase 1 の目玉となるリアルタイムゲーム。このissueではクライアント側を実装する。球を操作して迷路を進み、ゴール到達をサーバーに通知する。物理はクライアントで回し、順位の決定はサーバーに委ねる。

## スコープ

apps/web/src/games/tilt-maze に、Tilt Maze のクライアントを実装する。

### このissueでやること

- 10面の迷路
- 球の物理シミュレーション（クライアント側）
- VirtualPad による操作
- Canvas による描画
- ゴール到達のサーバーへの通知

### このissueでやらないこと

- サーバー側の順位確定（issue-13）
- 加速度センサー入力（オプション、後続）

## 設計方針

- RealtimeGame として実装する（ADR 0004）
- 入力は基盤共通の VirtualPad を使う（ADR 0018）
- 物理はクライアントで回し、ゴール到達を通知する。順位はサーバー受信順で確定するため、クライアントは決定権を持たない（ADR 0014）

## 受け入れ条件

- 球を操作して10面を進められる
- VirtualPad で操作できる
- ゴール到達がサーバーに通知される

## 段階

1. マニフェストを作り registry に登録する
2. 迷路と球の物理を実装する
3. VirtualPad 入力と Canvas 描画を繋ぐ
4. ゴール通知を実装する
