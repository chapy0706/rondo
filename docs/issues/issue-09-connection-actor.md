# issue-09: 接続アクター + レジストリ

## 背景

1接続を1つのプロセスとして扱う基盤を作る。接続の生存管理と、切断時の再接続猶予をここで扱う。単一接続を多重化する土台にもなる。

## スコープ

server/ に接続アクターと接続レジストリを実装する。

### このissueでやること

- connection_actor（1接続 = 1プロセス）
- 接続レジストリ
- 切断時の再接続猶予タイマーの骨格

### このissueでやらないこと

- ルームの状態管理（issue-10）

## 設計方針

- 接続をアクターとして扱い、状態をプロセスに閉じ込める（ADR 0006）
- 切断には10秒の再接続猶予を設ける（ADR 0013）
- 単一の WebSocket 接続を多重化する前提で設計する（ADR 0007）

## 受け入れ条件

- 接続ごとに接続アクターが立つ
- 切断時に再接続猶予タイマーが動く
- レジストリで接続を引ける

## 段階

1. connection_actor を実装する
2. 接続レジストリを実装する
3. 再接続猶予タイマーの骨格を入れる
