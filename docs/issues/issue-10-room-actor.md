# issue-10: ルームアクター + スーパーバイザー

## 背景

1ルームを1つのプロセスとして扱い、ルームの生成・参加・解散を担う。スーパーバイザーがルームの生成と監視を行い、障害からの回復を OTP の仕組みに委ねる。

## スコープ

server/ にルームアクターとスーパーバイザーを実装する。

### このissueでやること

- room_actor（1ルーム = 1プロセス）
- room_supervisor（生成・監視）
- ルームの作成・参加・解散
- 離脱確定と最小人数割れの判定

### このissueでやらないこと

- ゲーム固有の権威ロジック（issue-13）

## 設計方針

- ルームをアクターとして扱い、状態をプロセスに閉じ込める（ADR 0006）
- ルームは1ゲームの終了で解散する（ADR 0017）
- 再接続猶予を過ぎた離脱の確定と、最小人数割れでの終了を判定する（ADR 0013）

## 受け入れ条件

- ルームの生成・参加・解散ができる
- スーパーバイザーがルームを監視し、異常終了から回復する
- 最小人数を下回るとゲームが終了する

## 段階

1. room_actor を実装する
2. room_supervisor を実装する
3. 作成・参加・解散を実装する
4. 離脱確定・人数割れ判定を入れる
