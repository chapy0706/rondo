# issue-04: game-sdk

## 背景

game-sdk は、ゲームが基盤と話すための薄い開発キットである。契約の上に乗り、ゲームが基盤の内部を知らずに作れる状態を提供する。共通の入力手段もここに置く。

## スコープ

packages/game-sdk に、ゲーム開発用のフックと共通入力を実装する。

### このissueでやること

- ソロゲーム用フック（結果を基盤に返す）
- リアルタイムゲーム用フック（ルーム接続を扱う）
- 入力ポートと VirtualPad（基盤共通の仮想パッド）

### このissueでやらないこと

- 個別ゲームの実装（issue-07 以降）
- 加速度センサー（Accelerometer）の実装。オプションのため後続で対応

## 設計方針

- 入力は共通ポートとして抽象化し、ゲームは方向入力（x, y）だけを受け取る（ADR 0018）
- VirtualPad を Phase 1 の標準入力とする（ADR 0018）
- ソロゲーム用フックは RealTimePort を受け取らない（ADR 0004）

## 受け入れ条件

- ソロ / リアルタイム用フックが contracts の型に沿っている
- VirtualPad が方向入力（x, y）を供給する
- ゲーム本体が入力元を知らずに方向入力を受け取れる

## 段階

1. 入力ポートのインターフェースを定義する
2. VirtualPad を実装する
3. ソロゲーム用フックを実装する
4. リアルタイムゲーム用フックを実装する
