# issue-06: 基盤UI ゲーム選択画面

## 背景

ゲーム選択画面は、registry に集約されたマニフェストから自動生成される。ここがプラグイン機構の表側の要であり、ゲームを増やしても基盤側を触らずに選択肢が増える状態を実現する。メンバーが担当できる。

## スコープ

apps/web に、registry を読んで選択画面を組み立てる仕組みを実装する。

### このissueでやること

- registry からマニフェスト一覧を読み込む
- マニフェストからゲームカードを動的に生成する
- 選んだゲームの起動（ゲームホストへの導線）

### このissueでやらないこと

- registry に載るゲーム本体（issue-07 以降）
- ルーム・ロビー（issue-11）

## 設計方針

- ゲームはプラグインとして扱い、選択画面はマニフェストから自動生成する（ADR 0003）
- 基盤の UI・ルーティングにゲーム固有の記述を持たない

## 受け入れ条件

- registry にマニフェストを追加すると、選択画面の選択肢が増える
- 選択肢を増やす際に基盤側のコードを変更する必要がない
- 選んだゲームを起動できる

## 段階

1. registry の読み込みを実装する
2. マニフェストからカードを生成する
3. ゲーム起動の導線を繋ぐ
