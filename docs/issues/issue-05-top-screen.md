# issue-05: 基盤UI TOP画面

## 背景

TOP画面はプラットフォームの入口である。メンバーが担当できる、輪郭のはっきりした基盤UIのひとつ。リーダーが最小のプレースホルダーを先に置き、メンバーがそれを仕上げる前提とする。

## スコープ

apps/web に TOP画面を実装する。

### このissueでやること

- TOP画面のUI
- ゲーム選択画面への導線

### このissueでやらないこと

- ゲームロジック
- ルーム・ロビー

## 設計方針

- スマホの縦画面・タッチ操作を基本ターゲットとする（ADR 0012）
- 起動が速く軽いこと。気軽に立ち寄れる入口にする

## 受け入れ条件

- TOP画面が表示される
- ゲーム選択画面へ遷移できる
- 縦画面・タッチで操作できる

## 段階

1. リーダーがプレースホルダーを置く
2. メンバーが TOP画面のUIを仕上げる
3. 選択画面への遷移を繋ぐ
