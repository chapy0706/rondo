# issue-18: ソロゲーム Minesweeper

## 背景

ソロゲームの候補のひとつ。盤面を開きながら地雷を避け、周囲の数字を手がかりに安全なマスを推理する。

反射神経ではなく思考で進めるゲームであり、リアルタイム性がない分、実装を落ち着いて進められる。他の候補（2048、Snake）が方向入力の連続操作なのに対し、こちらはマスの選択という異なる操作性を持つ。

## スコープ

apps/web/src/games/minesweeper に、ソロゲームとして Minesweeper を実装する。

### このissueでやること

- 盤面の生成と地雷の配置
- マスを開く操作と、周囲の地雷数の表示
- 空きマスの連鎖展開
- フラグ立て
- 勝敗判定（地雷を開いたら失敗、地雷以外を全て開いたら成功）
- マニフェストの作成と registry への登録
- タッチによるマス選択の入力
- 結果の基盤への返却

### このissueでやらないこと

- リアルタイム要素・通信
- 記録の永続化（Phase 2）
- 難易度選択（盤面サイズは固定でよい）

## 設計方針

- SoloGame として実装し、リアルタイム接続を受け取らない（ADR 0004）
- 操作はマスのタッチ選択が中心となる。VirtualPad による方向入力は必須ではないが、方向入力でカーソルを動かす形にしてもよい。いずれの場合も入力元を game-sdk 経由で受け取り、独自に入力を触らない（ADR 0018）
- マニフェストで自己記述し、registry に登録する（ADR 0003）
- 盤面生成・連鎖展開・勝敗判定は判定ロジックであり、テストファーストで書く
- スマホの縦画面・タッチ操作を前提に、マスのサイズと余白を設計する（ADR 0012）
- 追加にあたり基盤側のコードには手を入れない

## 受け入れ条件

- ゲーム選択画面から起動して遊べる
- マスを開くと周囲の地雷数が表示され、空きマスが連鎖展開する
- フラグを立てられる
- 地雷を開くと失敗、地雷以外を全て開くと成功として終了し、結果が基盤に返る
- 盤面ロジックのテストが通る
- スマホの縦画面でマスが押しやすい大きさになっている
- 追加にあたり基盤側のコードを変更していない
- 1ゲームが2〜3分程度で終わる想定の盤面サイズになっている

## 段階

1. マニフェストを作り registry に登録する
2. 盤面生成と地雷配置をテストファーストで実装する
3. マスを開く操作と連鎖展開を実装する
4. フラグ立てと勝敗判定を実装する
5. タッチ入力と描画を繋ぐ（スマホの押しやすさを確認する）
6. 結果返却を繋ぐ
