# ADR 0018: 入力を基盤共通のポートとして抽象化する

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

Tilt Maze は傾き入力を使うが、加速度センサーは iOS で利用許可と HTTPS が必須で、機種差のハマりどころも多い。一方で、方向入力は Tilt Maze に限らず多くのゲームで使える。入力手段を個別ゲームに閉じるか、基盤で共通化するかを決める必要がある。

## 決定

入力を game-sdk 側の共通ポートとして抽象化する。ゲーム本体は方向入力（x, y）という契約だけを受け取り、それが仮想パッドから来るか加速度センサーから来るかを知らない。基盤は共通の仮想パッド（VirtualPad）を提供し、加速度センサー（Accelerometer）は利用できるゲームだけが追加で使うオプションとする。Phase 1 は仮想パッドを先に実装する。

## 理由

- 入力元を差し替え可能にすることで、ゲーム本体に手を入れずに入力手段を増やせる
- 仮想パッドは全端末で確実に動き、iOS の許可ダイアログや HTTPS 要件・機種差にハマらない
- 共通入力を基盤に置くことで、メンバーが作るソロゲームでも同じ入力をそのまま使える
- 加速度センサーは目玉として後から安全に追加できる

## 検討した代替案

- 加速度センサー版と仮想パッド版を個別ゲームごとに二重実装する案。入力まわりをゲーム数だけ作り直すことになり、死守ラインを遠ざけるため却下した
- 加速度センサーを Phase 1 の必須とする案。iOS 要件と機種差で発表前の時間を溶かす risk が高く、見送った

## 結果

- game-sdk に入力ポートと VirtualPad を置く
- Accelerometer はオプション実装とし、Phase 1 では仮想パッドを優先する
