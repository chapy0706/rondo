# ADR 0004: ゲームをソロ / リアルタイムの2層に分ける

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

rondo には、通信を必要としないソロゲーム（テトリス等）と、複数人がサーバーを介して同期するリアルタイムゲーム（Tilt Maze 等）の両方が並ぶ。すべてを同じ抽象で扱うと、通信の都合がソロゲームにまで漏れ出す。

## 決定

ゲームを SoloGame と RealtimeGame の2種類に明確に分ける。SoloGame はブラウザ内で完結し、リアルタイム接続を一切受け取らない。RealtimeGame だけがルーム接続（RealTimePort）を受け取る。

## 理由

- 必要なものだけを繋ぐ（UNIX 哲学）。ソロゲームは通信を知らなくてよい
- ソロゲームは基盤との接点がマニフェストとスコア返却だけになり、実装が軽い。数を稼ぎやすく、メンバーにも渡しやすい
- リアルタイムの複雑さを RealtimeGame の側に閉じ込められる

## 検討した代替案

- すべてのゲームを一律に RealtimeGame として扱う案。ソロゲームにも不要な接続概念が漏れ、単純なゲームが過剰に複雑になるため却下した

## 結果

- contracts に Game を基底とし、SoloGame と RealtimeGame を派生として定義する
- マニフェストの kind フィールド（solo / realtime）で種別を宣言する
