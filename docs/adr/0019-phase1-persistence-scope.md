# ADR 0019: Phase 1 の保存範囲は表示名のみとする

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

何を永続化し、何を揮発させるかを決める必要がある。ランキングや対戦履歴の永続化は Phase 2 の課題だが、何も保存しないとリロードで表示名すら消えてしまう。

## 決定

Phase 1 で端末に保存するのは、プレイヤーの固有情報（表示名程度）のみとする。ルーム状態・ゲーム進行・対戦結果などは揮発でよく、サーバー側もインメモリで保持する。永続データベースへの保存は Phase 2 で導入する。

## 理由

- 表示名だけを保持すれば、再訪時に名前を入れ直す手間がなくなる
- ゲーム進行やルーム状態は本来一時的なもので、永続化する必要がない
- 永続化の導入を Phase 2 に遅らせることで、Phase 1 のスコープを軽く保てる

## 検討した代替案

- 何も保存しない案。リロードのたびに表示名が消え、体験が悪いため却下した
- Phase 1 から対戦結果を永続化する案。データベースとスキーマ設計が前倒しになり、死守ラインを重くするため見送った

## 結果

- 表示名は端末に保持する（ADR 0015 と整合）
- リアルタイム基盤の状態はインメモリで持ち、永続化は Phase 2 の課題とする
