# ADR 0006: ルームと接続をアクターとして扱う

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

リアルタイム基盤では、多数のルームと接続が同時に存在し、それぞれが独立した状態（参加者、進行状況、接続の生存）を持つ。これらを共有状態として一括管理すると、競合と障害の波及が問題になる。

## 決定

1ルームを1つの OTP プロセス（ルームアクター）、1接続を1つの OTP プロセス（接続アクター）として扱う。ルームの生成・監視はスーパーバイザーが行う。

## 理由

- 状態をプロセスごとに閉じ込めることで、競合を構造的に避けられる
- 1つのルームや接続が異常終了しても、他のプロセスに波及しない
- スーパーバイザーによる監視で、障害からの回復を OTP の仕組みに委ねられる

## 検討した代替案

- 単一プロセスで全ルームの状態を保持する案。実装は単純だが、1つの不具合が全ルームを巻き込むため却下した

## 結果

- room_actor / connection_actor / room_supervisor を server に置く
- ルームの状態遷移ロジックはアクター内に閉じる
