# ADR 0011: モノレポ構成を採る

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

rondo は TypeScript（フロント・契約・SDK・ゲーム）と Gleam（リアルタイム基盤）という2言語で構成される。これらを別リポジトリに分けるか、ひとつのリポジトリにまとめるかを決める必要がある。

## 決定

モノレポ構成を採る。TypeScript 側は pnpm workspaces で packages / apps を束ね、Gleam サーバーは同じリポジトリの server/ に併置する。

## 理由

- 契約（contracts）をフロントとゲームが共有するため、同一リポジトリだと型の共有が容易になる
- フロントと Gleam の変更を1つのコミット・1つの CI で扱える
- 個人・少人数開発では、リポジトリを跨ぐ運用コストの方が重い

## 検討した代替案

- フロントと Gleam を別リポジトリに分ける案。契約の共有とバージョン整合の管理が煩雑になるため却下した

## 結果

- pnpm-workspace.yaml で packages / apps を定義する
- server/ に Gleam プロジェクトを置き、Makefile から両者を横断的に操作する
