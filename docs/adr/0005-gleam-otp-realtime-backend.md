# ADR 0005: リアルタイム基盤に Gleam/OTP + mist を採用する

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

リアルタイムゲームには、ルームと接続を多数同時に扱う WebSocket サーバーが必要になる。各ルーム・各接続が独立した状態を持ち、片方の障害が他に波及しない構造が望ましい。

## 決定

リアルタイム基盤のバックエンドに Gleam/OTP + mist を採用する。nagomi-ws で確立した Gleam/OTP + mist の構成を土台として再利用する。

## 理由

- OTP のアクターモデルは「1ルーム = 1プロセス」「1接続 = 1プロセス」という分割に自然に対応し、マルチプレイヤーのルーム管理に本質的に向いている
- 軽量プロセスと障害分離により、一部のルームが落ちても全体が止まらない
- nagomi-ws で得た Gleam/OTP + mist の知見をそのまま活かせる
- 型のある関数型言語で、低レイヤー志向とも噛み合う

## 検討した代替案

- かくれんぼ案で想定していた Node.js + Socket.io。手早く書けるが、ルーム・接続の障害分離やプロセス単位の状態管理で OTP の優位に及ばないため却下した

## 結果

- かくれんぼ案にあった Socket.io 構成は破棄し、Gleam/OTP に統一する
- サーバーは server/ に Gleam プロジェクトとして配置する
