# ADR 0008: サーバー権威の量はゲームごとに決める

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

リアルタイムゲームでは、勝敗確定や不正防止のためにサーバー側の権威判定が必要になることがある。一方で、すべてをサーバーで判定すると通信量と遅延が増す。必要な権威の量はゲームの性質によって大きく異なる。

## 決定

基盤は「サーバー権威を載せられる箱」だけを用意する。どこまでをサーバー権威とし、どこからをクライアント表現に委ねるかは、ゲームごとに個別の ADR で決める。

## 理由

- ゲームによって必要な権威の量が異なるため、基盤で一律に決めると過不足が出る
- 基盤を薄く保つことで、ゲーム側の判断の自由を残せる
- nagomi-ws の「薄い中継点」の思想を引き継ぎつつ、必要なゲームだけが権威を足せる

## 検討した代替案

- 基盤がすべてのゲームの状態を権威的に管理する案。汎用性を欠き、ソロゲームにも無用な権威概念が漏れるため却下した
- 完全にクライアント任せにする案。不正防止が必要なゲームで成立しないため却下した

## 結果

- server/src/games/<id>/ にゲーム固有の権威ロジックを置ける構造にする
- 個別ゲームの権威方針は、そのゲームの ADR で別途定める
