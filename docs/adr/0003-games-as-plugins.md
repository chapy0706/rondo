# ADR 0003: ゲームをプラグインとして扱う

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

プラットフォーム上で複数のゲームを扱い、今後も数を増やしていきたい。ゲームを追加するたびに基盤側のコードを書き換える構造だと、追加のたびに基盤が壊れる risk があり、拡張が重くなる。

## 決定

各ゲームは自身を記述したマニフェスト（GameManifest）を持つ。基盤は registry でマニフェストを集約し、ゲーム選択画面を自動生成する。ゲームの追加は、ゲーム本体とマニフェストの追加、および registry への登録だけで完結させ、基盤の UI・ルーティング・接続管理には手を入れない。

## 理由

- 拡張に開き、修正に閉じる。新しいゲームの追加が基盤に波及しない
- 「ゲームはプラグインとして追加でき、基盤はゲームを知らない」という構造そのものが、設計の説明として強い
- ゲームを作る人（メンバーを含む）が、基盤の内部を理解せずに作業できる

## 検討した代替案

- 選択画面に各ゲームをハードコードする案。ゲーム追加のたびに基盤側の改修が必要になり、責務が漏れるため却下した

## 結果

- registry だけが全ゲームの一覧を知る単一の集約点になる
- 選択画面はマニフェストから動的に組み立てられる
