# ADR 0012: スマホ縦画面を基本ターゲットとする

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

rondo は「ちょっと集まって少し遊ぶ」軽い体験を核に置く。利用シーンとして、手元のスマホで気軽に遊ぶことを最も重視する。設計判断の優先順位を定めるため、基本ターゲットを明確にしておく。

## 決定

スマホの縦画面・タッチ操作を基本ターゲットとする。PC や横画面は副とし、設計判断が衝突した場合はスマホ体験を優先する。ゲーム単位で必要な画面の向きを宣言できる余地は残す。

## 理由

- 気軽さを核とするコンセプトに、最も手近なデバイスであるスマホが合う
- ターゲットを明確にすることで、UI とレイアウトの判断がぶれない
- 具体的な入力手段（仮想パッド・加速度）は ADR 0018 に委ね、本 ADR は画面と操作の前提だけを定める

## 検討した代替案

- PC を主ターゲットとする案。気軽さの核から外れ、利用シーンと合わないため却下した

## 結果

- UI は縦画面・タッチ前提で設計する
- マニフェストでゲームごとに推奨する画面の向きを宣言できるようにする
