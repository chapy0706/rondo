# ADR 0016: ルーム参加は一覧からの参加または新規作成とする

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

複数人で同じルームに入る導線を決める必要がある。部屋コード共有、URL 共有、ランダムマッチングなど複数の方式がある。

## 決定

立ち上がっているルームを一覧で表示し、そこから参加する。参加したいルームがなければ新規作成する。ランダムマッチングは Phase 1 では作らない。

## 理由

- 一覧から選ぶ・なければ作る、という流れは直感的で、追加の共有操作を必要としない
- ランダムマッチングは人数が揃わないと成立せず、小規模な利用シーンに合わない
- 実装が単純で、Phase 1 の死守ラインに収まる

## 検討した代替案

- 部屋コード／URL 共有のみの案。少人数の身内利用には合うが、一覧の手軽さに劣るため一次案から外した（将来の追加余地は残す）
- ランダムマッチング。母数が必要で Phase 1 では成立しないため見送った

## 結果

- ロビーにルーム一覧と新規作成を置く
- ルーム一覧の取得は単一 WebSocket 接続（ADR 0007）を通じて行う
