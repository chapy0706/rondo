# ADR 0014: 同期はサーバー受信順を権威とする

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

Tilt Maze のようなリアルタイムゲームでは、ゴール到達順などの勝敗を確定する基準が必要になる。クライアントが打刻したタイムスタンプは、端末ごとの時計のずれや詐称の余地があり、そのままでは信頼できない。

## 決定

順位や勝敗は、サーバーがイベント（ゴール到達等）を受信した順を正とする。クライアント側のタイムスタンプやラグのタイミングは採用しない。

## 理由

- 端末の時計に依存せず、単一のサーバー時計で一貫した判定ができる
- クライアントの詐称を構造的に避けられる
- 回線の速い遅いによる多少の有利不利は、Phase 1 では許容する割り切りとする

## 検討した代替案

- クライアントのタイムスタンプを採用する案。時計のずれと詐称の余地があるため却下した
- ラグを補正して公平を期す案。実装が重く、発表会向けの規模に過剰なため見送った

## 結果

- サーバーは受信時刻でイベントを順序づける
- クライアントは到達を通知するだけで、順位の決定権を持たない
