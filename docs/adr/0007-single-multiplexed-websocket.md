# ADR 0007: WebSocket 接続は1本で多重化する

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

クライアントは、ロビー・ルーム・ゲーム進行など複数の文脈でサーバーと通信する。文脈ごとに接続を張ると、接続管理が煩雑になり、モバイル環境では接続数そのものが負担になる。

## 決定

クライアントとサーバーの間の WebSocket 接続は1本とする。メッセージに種別（gameType / roomId 等）を載せ、サーバー・クライアント双方でルーティングして多重化する。

## 理由

- 接続管理が単純になり、再接続処理も1本だけ面倒を見ればよい
- モバイル回線で複数接続を維持する負担を避けられる
- メッセージ契約に種別を持たせることで、多重化を型で扱える

## 検討した代替案

- 文脈ごとに接続を分ける案。接続のライフサイクル管理が文脈の数だけ増えるため却下した

## 結果

- protocol / messages のメッセージ型に、ルーティングに必要な種別を含める
- クライアント側の WebSocketAdapter が単一接続を多重化して各機能に配る
