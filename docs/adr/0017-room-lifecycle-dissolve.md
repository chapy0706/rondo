# ADR 0017: ルームは1ゲームの終了で解散する

- ステータス: 受理
- 日付: 2026-06-24

## 文脈

1ゲームが終わった後、ルームを維持して同じメンバーで次のゲームに移れるようにするか、解散するかを決める必要がある。これは rondo の「巡る」というコンセプトの解釈にも関わる。

## 決定

ルームは1ゲームのクリア（終了）をもって解散する。同じメンバーで連続して別ゲームに移る機能は持たない。rondo の「巡る」は、同じ仲間で巡ることではなく、多数のゲームをさまざまな組み合わせで選び巡れる選択肢の多さとして表現する。

## 理由

- ルームの状態遷移が「作る → 遊ぶ → 解散」と単純になり、実装が軽い
- 「巡る」をプラットフォーム全体（一人ひとりが選び直しを繰り返す）で表現する解釈が、プラグインで増えていくゲーム構造と噛み合う
- 連続プレイの導線は Phase 1 の死守ラインには不要

## 検討した代替案

- 1ゲーム終了後に同じメンバーでゲーム選択へ戻る案。ルームのライフサイクルにループが加わり設計が増えるため、Phase 1 では見送った（将来の候補としては残す）

## 結果

- ルームアクターはゲーム終了で破棄される
- design-spec のコンセプト章に「巡る」の定義（選択肢の多さ）を明記する
